# Student Comment Support Pack

Use this skill for student comments, teacher remarks, and individualized support suggestions.

## What this skill is for

This skill is optimized for:

- student report comments
- term-end comments
- strengths + improvement direction summaries
- learning support notes
- teacher-facing support recommendations
- individualized support drafts

## Workflow

1. Extract the student profile from the user's brief:
   - grade
   - student name if given
   - strengths
   - observed difficulties
   - tone expectation
   - support goal
2. If the brief already includes strengths and challenges, do not ask for them again.
3. Distinguish between:
   - praise language for parents/students
   - practical support notes for teachers
4. If generating a student comment:
   - write with warmth
   - avoid template-like praise
   - include a believable next-step direction
5. If generating support suggestions:
   - make them concrete
   - keep them classroom-usable
   - avoid vague slogans

## Output modes

Choose the right mode based on the task:

- term comment
- concise class teacher comment
- teacher-facing support note
- student support plan summary

## Style rules

- Sound like a real Chinese teacher, not a performance review tool.
- Keep praise grounded in observable behavior.
- Keep improvement advice clear but not harsh.
- Prefer “具体表现 + 发展方向” over empty adjectives.
- For support plans, focus on what can be done in class or at home in practical terms.

## Boundaries

- Do not fabricate achievements or problems not supported by the brief.
- Do not produce shaming, humiliating, or overly negative wording.
- Do not write comments that sound identical across students.
