# Claw Mart 教师 Skill 上架方案

更新时间：2026-04-04

## 1. 目标

把现有“教师真实工作流经验”包装成可上架、可收款、可持续迭代的 Claw Mart skill。

核心目标不是卖一个“网页工具站”，而是卖：

- 面向 OpenClaw / agent 用户的教师工作流能力包
- 面向中国 K-12 教学场景的高价值单点 skill
- 可复制、可扩展、可逐步涨价的数字商品

## 2. 市场结论

### 2.1 这个市场值不值得试

值得试，而且适合用“小而实”的产品切入。

观察到的公开信号：

- Claw Mart 首页公开写了：
  - `2,000+ Listings`
  - `250+ creators`
  - `$100,000+ creator earnings`
- 平台支持个人创作者直接发布技能与角色
- 平台支持免费与付费商品，并提供 Creator API

参考：

- [Claw Mart 首页](https://www.shopclawmart.com/)
- [Claw Mart About](https://www.shopclawmart.com/about)
- [How to Publish Your Skill to Claw Mart](https://www.shopclawmart.com/blog/how-to-publish-skill-to-claw-mart)

### 2.2 卖得好的是什么

卖得好的，不是“行业模板大全”，而是：

- 一眼能看懂价值的单能力包
- 让 agent 立刻变强的 skill / persona
- 有明确结果、有实操感、有维护感的商品

公开 listing 示例：

- `Coding Agent Loops`：`1,351 sold`
- `Felix`：`1,019 sold`
- `AI Agent Quick Start Template`：`406 sold`

这些商品的共性：

- 标题非常直接
- tagline 讲结果，不讲抽象概念
- About 讲真实使用场景和交付内容
- 经常强调 battle-tested、playbooks、tool config、step-by-step setup

参考：

- [Coding Agent Loops](https://www.shopclawmart.com/listings/coding-sessions-b5ce6e4b)
- [Felix](https://www.shopclawmart.com/listings/felix-04f42dee)
- [AI Agent Quick Start Template](https://www.shopclawmart.com/listings/ai-agent-quick-start-template-6c419078)
- [Agent Ops Playbook](https://www.shopclawmart.com/listings/agent-ops-playbook-bf0d8a2e)

### 2.3 教师方向有没有空位

有，而且很明显。

当前公开 listings 和站内关键词没有看到明确的：

- teacher
- lesson
- classroom
- education
- school

也就是说：

- 竞争小
- 市场教育成本更高
- 但如果切口够清晰，反而更容易形成差异化

## 3. 我们应该怎么卖

### 3.1 不卖什么

不要卖成：

- “另一个教育工具站”
- “某某平台平替”
- “一堆工具集合页”
- “和 MagicSchool 很像的老师平台”

原因：

- 容易撞表达
- 容易撞产品结构
- 容易引发不必要的品牌联想
- 在 Claw Mart 这种市场里也不利于成交

### 3.2 应该卖什么

卖成：

- 一条清晰工作流
- 一个明确痛点
- 一个 agent 能安装后立刻获得的能力

正确表达方式：

- `Parent Communication for Chinese Teachers`
- `Student Comment + Support Pack`
- `Lesson Pack Generator`

也就是卖：

- 教师工作流 skill
- 教学场景能力包
- 真实中国 K-12 场景 playbook

## 4. 第一批 SKU 建议

### 4.1 免费引流款

#### 商品名

`Parent Communication for Chinese Teachers`

#### 卖点

- 一句话输入真实班级情况
- 拆出沟通对象、问题、语气、沟通目的
- 生成适合中国教师语境的家校沟通文本
- 避免太模板化、太生硬、太像客服文案

#### 为什么适合做免费

- 场景最容易理解
- 购买门槛最低
- 最容易让人“一装就知道有用”
- 可以作为后续高价值 skill 的漏斗入口

#### 定价建议

- `Free`

### 4.2 低价付费款

#### 商品名

`Student Comment + Support Pack`

#### 卖点

- 学生综合素质评语
- 个性化学习支持建议
- 语义归并，避免重复追问
- 输出更像班主任/任课老师真实会写的内容

#### 适合的购买人群

- 想快速试老师场景 skill 的人
- 做教育 agent、校园工具、学习支持工具的人

#### 定价建议

- `$9`
- 如果初期反馈好，可提到 `$19`

### 4.3 主打付费款

#### 商品名

`Lesson Pack Generator for Chinese Classrooms`

#### 卖点

- 从一句种子输入开始
- 拆解硬约束与软建议
- 生成备课相关材料
- 更适合中国 K-12 课堂语境
- 可用于教案、导入、练习、作业等连续资产生成

#### 适合的购买人群

- 做教育 agent 的开发者
- 想要真实中文教学 workflow 的 agent 用户
- 想快速切入教师场景的人

#### 定价建议

- `$19` 起
- 如果交付结构足够完整，可到 `$29`

## 5. 避嫌与差异化原则

### 5.1 命名避嫌

不要出现：

- MagicSchool
- 任何相近命名
- “某某替代品”
- “某某平替”

### 5.2 文案避嫌

不要写：

- 类似某平台
- 替代某平台
- 复刻某平台功能

要写：

- built for Chinese K-12 classroom workflows
- based on real teacher communication and lesson prep scenarios
- optimized for natural-language intake and classroom-ready outputs

### 5.3 结构避嫌

不要把商品包装成某平台的镜像分类。

不要直接复用：

- 类别名
- 页面模块名
- 介绍口吻
- 卡片文案结构

### 5.4 方法论差异化

我们的独立方法论应该被强调：

- 种子输入
- 语义拆分
- 硬约束 / 软建议
- 缺口驱动追问
- 节点式工作流
- 结果编辑器
- 局部改写与局部重算

## 6. 商品详情页怎么写

### 6.1 标题公式

`结果 + 适用对象 + 场景`

例如：

- `Parent Communication for Chinese Teachers`
- `Lesson Pack Generator for Chinese Classrooms`
- `Student Comment + Support Pack`

### 6.2 tagline 公式

`一句话说清装上以后 agent 会获得什么能力`

例如：

- Turn one messy teacher note into calm, usable parent communication.
- Generate classroom-ready lesson assets from one natural-language brief.
- Write warmer, more realistic student comments and support plans for Chinese K-12 contexts.

### 6.3 About 结构

推荐统一 5 段：

1. 这东西解决什么问题
2. 它适合谁
3. 输入是什么
4. 输出是什么
5. 边界是什么

### 6.4 不要写的东西

不要写：

- “AI 很强大”
- “多模型支持”
- “最先进”
- “一切都能做”

要写：

- 场景
- 输入
- 输出
- 稳定性
- 真实使用方式

## 7. 打包结构建议

每个上架 skill 至少包含：

- `SKILL.md`
- `README.md`
- `examples/`
- `templates/`

建议结构：

```text
skill-name/
  SKILL.md
  README.md
  examples/
    example-1.md
    example-2.md
  templates/
    parent-communication.md
    student-comment.md
  assets/
    cover.png
```

如果是主打商品，再加：

- `VERSION.md`
- `CHANGELOG.md`
- `INSTALL.md`

## 8. 我们的包装方向

### 8.1 第一阶段

目标：先拿到第一笔收入和第一批用户反馈

动作：

- 上一个免费引流款
- 上一个低价付费款
- 不超过 3 个 listing
- 先验证点击率、转化率、下载后反馈

### 8.2 第二阶段

目标：把“教师 skill”做成有复购的系列

动作：

- 把单点 skill 组合成 bundle
- 做更高价的 workflow pack
- 增加更新日志和版本历史

### 8.3 第三阶段

目标：做 persona 或完整 teacher operator

前提：

- 单 skill 已有验证
- 用户知道你们不是在卖 prompt，而是在卖能力系统

## 9. 立刻可执行的路线

### 第一周

- 定 3 个 SKU
- 完成命名与定位
- 写 listing 文案初稿
- 做封面图和交付结构

### 第二周

- 上线免费款 + 一个付费款
- 观察浏览、下载、转化
- 根据反馈修标题、tagline、封面、about

### 第三周

- 上主打付费款
- 开始做版本更新和用户反馈沉淀

## 10. 当前推荐执行顺序

1. 先做 `Parent Communication for Chinese Teachers`
2. 再做 `Student Comment + Support Pack`
3. 最后做 `Lesson Pack Generator for Chinese Classrooms`

原因：

- 从最容易理解的场景切入
- 从最容易展示价值的场景切入
- 从最容易和现有平台形成差异化的场景切入

## 11. 下一步建议

优先做这三件事：

1. 给第一批 3 个 listing 写完整英文文案
2. 设计每个 listing 的交付文件结构
3. 先做一个免费款上架验证市场
